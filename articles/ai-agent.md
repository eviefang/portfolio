从 AI 对话到 Workflow 到 Agent，能力演进的方向很清晰。但 2025-2026 年，围绕这条演进线涌现了大量产品——Manus、OpenClaw、Coze、Dify、飞书秒搭——定位高度重叠，即使是持续关注 AI 的从业者也很难在 30 秒内说清楚每个产品该用在什么场景。

## 演进的四个阶段

第一阶段是纯对话：用户把需求说清楚，AI 返回文字回答，核心能力是 prompt。

第二阶段是 Workflow：把任务拆成固定节点，告诉 AI 先做什么再做什么。好处是稳定可控，坏处是人类在替 AI 做规划，AI 只负责执行。Coze、Dify 做的就是这件事。

第三阶段是 Agent：AI 自己规划、执行、根据中间结果调整、再执行，形成自主决策循环。Manus 是这个方向的代表——用户给一个任务，它自己搜索、整理、生成可交付的成果物。

第四阶段是多 Agent 协作：多个专精 Agent 各自完成擅长的子任务，目前更多停留在概念阶段。

## 真正的问题：中间层的价值在被挤压

Coze 这类平台本质上是图形化的低代码 Agent 搭建工具——降低理解成本，让用户快速上手。但随着大模型自身能力的飞速增强，一个尴尬的事实浮现：GPT、Claude、Gemini 本身就在变成通用 Agent。

我现在做一个网页，直接和 Claude 对话就能生成，不需要去 Coze 上拖拽节点搭流程。我在阿里实习时搭建的 Agent，也是直接操作大模型完成的——告诉它什么时候自己思考、自己规划、用循环去执行任务。当大模型自己就能写代码、调工具、做规划的时候，中间那层"低代码搭建平台"的价值正在被快速压缩。

> 这些平台面临的困境是：门槛降得再低，也低不过"直接和大模型对话"。而它们提供的额外价值——流程可视化、节点编排——对于真正复杂的任务又不够用，对于简单任务又多此一举。

## 产品边界模糊：每个都在做别人的事

让用户更困惑的是，这些产品都没有停留在自己最初的阶段。Coze 和 Dify 起步于 Workflow，但现在也能搭 Agent、写 Skill、生成网页应用；Manus 起步于 Agent 任务执行，但现在官网也能做网页、移动端 App。每个产品都在拼命扩展边界，试图覆盖所有场景。

结果就是：当用户想做一个网页应用，Coze 能做，Manus 能做，Claude 也能做——到底选哪个？没有任何一个产品给出过清晰的回答。用户的选择依据被迫退化为"哪个我恰好用过""哪个这次运气好效果不错"，这对一个成熟赛道来说是不正常的。

## Manus 的"一次性交付"也有隐患

Manus 相比 Coze、Dify 的进步在于：用户不需要自己搭流程，给一个任务它自主完成，直接交付成果物。但这种模式在复杂任务上有一个天然矛盾——你没法在执行过程中介入调整。如果一个任务跑了 20 分钟交付的结果不满意，你只能重新来一次。

> 这本质上是把"人机协作"变成了"人机赌博"：简单任务一次搞定很爽，复杂任务赌中的概率就会下降。反而是 Claude 这种对话模式，你可以随时查看中间结果、随时纠偏方向，在复杂任务上的可控性更高。Manus 赌的是模型能力足够强到一次做对，但现阶段显然还没到那个水平。

## OpenClaw 的真正突破：不是能力，是范式

OpenClaw 之所以能在这个阶段爆火，不是因为它在 Agent 能力上有多大的提升，而是它改变了产品和用户的关系。

之前所有产品——不管是 ChatGPT、Coze 还是 Manus——面对的都是大众用户。为了让大部分人满意，它们只能把能力做得尽可能通用，去覆盖最大公约数的任务场景。但通用就意味着不可能在任何一个方向上做到极致，永远是 70 分的体验。

OpenClaw 换了一个思路：它不给你一个成品产品，而是给你一个框架——允许你部署在自己的电脑上，让它学习你的思考方式、你喜欢的交流方式、你自己的文件和知识库。本质上你是在花时间喂养一个专属于你个人的私人 Agent。

这就是它和所有其他产品最本质的区别：其他产品是"一个产品服务所有人"，OpenClaw 是"一个框架让每个人养自己的 AI"。一个为你个人而生的 Agent，处理你自己的专属任务怎么会不好用？只要你教它教得足够多、给它的能力足够强，它可以变成任何领域的专家。

当然代价也很明确：技术门槛高、时间投入大、需要持续喂养。这决定了它短期内只属于有技术能力且愿意投入的用户。

## 为什么我暂不部署 OpenClaw

研究完之后我评估了自己是否应该部署，结论是当前阶段不值得。

第一是安全风险。OpenClaw 仍处于发展阶段，隐私性和安全性未经充分验证。部署在工作电脑上容易导致 API 密钥和隐私文件泄露，而我缺乏专用设备进行安全隔离。

第二是经济成本。要发挥全部能力需要接入 Claude、Gemini、GPT 等模型，都需要订阅或按量付费，调教成本过高。

第三是功能受限。国内的云端部署方案只能使用单一厂商模型，无法发挥 OpenClaw 最大的价值——个人定制化。云端部署也无法自动获取本地工作习惯和文件信息，相比本地部署功能大幅缩水。

## 我的判断

AI Agent 赛道目前的核心瓶颈不是技术能力，而是产品定位的清晰度。每个产品都在往全能方向扩展，结果是功能高度重叠但各自都不够深入，没有一个产品能让用户在 30 秒内明白"我遇到这个问题就该用你"。

这个赛道最终的分化可能不是按"谁的功能更多"来分，而是按"人机协作的深度"来分：一端是 Manus 式的全托管（给任务等结果），另一端是 Claude 式的实时协作（边做边调），中间是 Coze/Dify 这类搭建平台。全托管适合简单标准化任务，实时协作适合复杂创造性任务，而中间层的低代码平台正在被两端同时挤压。

> 长期来看，大模型自身能力的增强会持续压缩中间层的空间。OpenClaw 的路线——本地部署、深度个人化——是少数能避开这种挤压的方向，因为它做的不是"比大模型更强"，而是"比大模型更懂你"。它指向的方向——从通用智能体到私人定制化——可能是 AI Agent 赛道真正的终局。
